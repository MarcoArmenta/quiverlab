# quiverlab-web (Plan 09) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A public web GUI (`webapp/`) that lets algebraists build a finite-dimensional algebra from the library's family catalog, compute invariants on the server, read results with rendered mathematics, and download artifacts (JSON, worked-steps PDF, TikZ) — with no accounts and no user code execution.

**Architecture:** A FastAPI app serves server-rendered pages (Jinja2 + KaTeX + vanilla JS) and a JSON API. Small computations run synchronously ("instant tier"); larger ones become rows in a single SQLite jobs table (WAL mode) consumed by a pool of single-job worker processes, each memory- and thread-capped. All library access is funnelled through one adapter module so signature drift against the finished library is reconciled in exactly one place. Deployment is a `docker-compose` stack (app + worker + Caddy-with-automatic-TLS) targeting a DRAC Arbutus persistent instance, but host-generic.

**Tech Stack:** Python ≥ 3.10; FastAPI, Uvicorn, Jinja2, Pydantic v2, `python-ulid`; stdlib `sqlite3` (WAL) as the queue; stdlib `resource` for rlimits; KaTeX (vendored static asset) for math rendering; Caddy for TLS/reverse proxy; Docker Compose for deployment. No Node toolchain, no Redis, no message broker.

**Prerequisite:** This plan is executed **after Plans 03–08** — the library is feature-complete and released (families catalog via `zoo()`/`families()`, all public invariants of spec §3.5, and the Plan-07 worked-steps PDF renderer all exist). The spec is `docs/specs/2026-07-18-quiverlab-web-design.md` (approved). The web app depends on the library only through its **documented public API** (spec §3), never `quiverlab.engine` or other internals.

## Global Constraints

- **Repo layout:** everything lands under `webapp/` in this repo. Nothing in `webapp/` may be imported by `src/quiverlab/` — the dependency arrow points one way (app → library).
- **Dependency isolation:** web dependencies go in a `[web]` optional extra in `pyproject.toml`. `pip install quiverlab` stays dependency-light (numpy, sympy, matplotlib only); `pip install quiverlab[web]` adds the server stack. A test asserts no top-level `quiverlab` import pulls in FastAPI.
- **No user code execution, ever.** Inputs are a family id + typed parameters + a field spec (+ v2 relation strings, which go only through the library parser). No `eval`, no `exec`, no `pickle.loads` of user data, no importing user-named modules. A security test asserts this.
- **Exactness is never softened.** The library's loud errors (`ExactnessError`, `RelationError`, `AdmissibilityError`, `DepthLimitError`, `FieldError`, `NotFiniteDimensionalError`) surface verbatim (type name + message) to the user; the app adds no float path and no silent fallback.
- **Single library boundary:** all calls into `quiverlab` live in `webapp/server/library_api.py`. No other module imports `quiverlab`. This is the one file that absorbs any public-API signature drift at implementation time.
- **Venv:** ALWAYS `/Users/marco/Desktop/HomologicalNetworks/quiverlab/.venv/bin/python` (system python is 3.8). Install the app with `... -m pip install -e ".[web,dev]"` at Task 1.
- **Resource discipline:** every worker process sets `NUMBA_NUM_THREADS=2` and `OMP_NUM_THREADS=2` before importing quiverlab, and applies an address-space rlimit. No process spawns unbounded children. Test commands also export those two env vars. ≤5 parallel agents during execution of this plan.
- **Async safety:** the FastAPI event loop must never run a library computation inline. Instant-tier compute runs in a thread pool with a wall-clock watchdog; anything heavier is a job. No blocking call on the event loop thread.
- **Commits:** conventional commits; green suite at every commit; end every commit message with the trailer:

- **Testing:** pytest; FastAPI `TestClient` for API/pages; a real subprocess worker for the end-to-end job test. Run from repo root: `NUMBA_NUM_THREADS=2 OMP_NUM_THREADS=2 /Users/marco/Desktop/HomologicalNetworks/quiverlab/.venv/bin/python -m pytest -q webapp/tests`.

Write the boundary module. Full structure (the invariant dispatch table maps request tokens to public methods of spec §3.5; the `_parse_degree_range` helper parses `"0..6"`; any non-integer degree bound raises `ExactnessError` by delegating to the library or by an explicit exact-int check):

```python
from __future__ import annotations

from dataclasses import dataclass, asdict
from pathlib import Path
from typing import Any, Literal

import quiverlab
from quiverlab import CC, GF  # public field constructors (spec §3.2)


@dataclass
class ParamSpec:
    name: str
    kind: Literal["int", "int_list", "str", "bool"]
    default: Any
    constraints: dict


@dataclass
class FamilyInfo:
    key: str
    label: str
    params: list[ParamSpec]
    supports_fields: list[str]
    doc: str

    def to_dict(self) -> dict:
        d = asdict(self)
        return d


def library_version() -> str:
    return quiverlab.__version__


def field_from_spec(field: dict):
    kind = field["kind"]
    if kind == "CC":
        return CC
    if kind == "GF":
        p = int(field["p"])
        n = int(field.get("n", 1))
        if p <= 1 or n < 1:
            raise ValueError("GF requires prime p>1 and n>=1")
        return GF(p ** n)
    raise ValueError(f"unknown field kind {kind!r}")


# The catalog is generated from the library's families() discoverability
# surface (spec §3.4). This registry maps each family key to its public
# constructor and declares its parameters. It is validated at import against
# the live library in list_families(); a family the library no longer exposes
# is dropped with a recorded note rather than crashing the app.
_FAMILY_REGISTRY: dict[str, dict] = {
    "truncated_polynomial": {
        "label": "Truncated polynomial  k[x]/(x^n)",
        "params": [ParamSpec("n", "int", 3, {"min": 2, "max": 200})],
    },
    "linear_path_algebra": {
        "label": "Linear path algebra  A_n (linear orientation)",
        "params": [ParamSpec("n", "int", 3, {"min": 1, "max": 200})],
    },
    # Remaining spec §3.4 families are registered here at implementation time
    # (Nakayama, PathAlgebra(Dynkin), RadicalSquareZero, QuantumCI, Exterior,
    #  TrivialExtension, ...) once their released public signatures are known.
}


def _resolve_constructor(key: str):
    # Import the public constructor by the library's documented name.
    # Reconciled at implementation time against the released API (spec §3.4).
    return getattr(quiverlab, _FAMILY_REGISTRY[key].get("ctor", key))


def list_families() -> list[FamilyInfo]:
    out: list[FamilyInfo] = []
    for key, meta in _FAMILY_REGISTRY.items():
        try:
            ctor = _resolve_constructor(key)
        except AttributeError:
            continue  # library no longer exposes it; skip, do not crash
        out.append(FamilyInfo(
            key=key,
            label=meta["label"],
            params=meta["params"],
            supports_fields=["CC", "GF"],
            doc=(ctor.__doc__ or "").strip().split("\n")[0],
        ))
    return out


def build_algebra(spec: dict):
    if spec.get("kind") != "family":
        raise ValueError("v1 supports only kind='family'")
    key = spec["family"]
    if key not in _FAMILY_REGISTRY:
        raise ValueError(f"unknown family {key!r}")
    field = field_from_spec(spec["field"])
    ctor = _resolve_constructor(key)
    params = spec.get("params", {})
    # Family constructors take the field as an explicit keyword (spec §3.4).
    return ctor(**params, field=field)


def _parse_degree_range(token: str) -> tuple[str, int]:
    # "hh_cohomology:0..6" -> ("hh_cohomology", 6). Bound must be an exact int;
    # a non-integer bound is an exactness violation and is raised as such.
    name, _, rng = token.partition(":")
    if not rng:
        return name, 0
    _, _, top = rng.partition("..")
    try:
        n = int(top)
    except ValueError:
        from quiverlab import ExactnessError
        raise ExactnessError(f"degree bound must be an exact integer, got {top!r}")
    return name, n


def run_invariants(algebra, requests: list[str], want_pdf: bool, out_dir: Path) -> dict:
    result: dict = {"result_schema": 1, "invariants": [], "unavailable": []}
    for token in requests:
        name, degree = _parse_degree_range(token)
        if name == "hh_cohomology":
            table = algebra.hochschild_cohomology(degree, verbose=want_pdf)
            result["hh_cohomology"] = {"dims": list(table.dims), "degree": degree}
        elif name == "hh_homology":
            table = algebra.hochschild_homology(degree, verbose=want_pdf)
            result["hh_homology"] = {"dims": list(table.dims), "degree": degree}
        elif name == "coxeter_polynomial":
            result["coxeter_polynomial"] = str(algebra.coxeter_polynomial())
        elif name == "cartan":
            result["cartan"] = [[str(x) for x in row] for row in algebra.cartan_matrix()]
        elif name == "global_dimension":
            result["global_dimension"] = str(algebra.global_dimension())
        elif name == "center":
            result["center_dim"] = int(algebra.center().dimension())
        else:
            result["unavailable"].append(name)
        result["invariants"].append(name)
    if want_pdf:
        # Plan-07 worked-steps traces are written by the library to its trace
        # directory; move/copy the produced PDF(s) into out_dir. The exact
        # trace-path API is reconciled at implementation time (spec §3.8).
        _collect_traces(out_dir)
    return result

def reproduction_snippet(spec: dict) -> str:
```
